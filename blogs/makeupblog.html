    个人博客搭建---vuepress+githubPage

    ShowedMaker 5/5/2023 个人博客

    哈喽，这是一期关于用 Vuepress 搭建博客并将它部署到 Github Pages 上的文章，参考了网上的很多文章，资料。今天刚刚自己写好，趁着熟悉，在这里记录一下。

    # 何为 Vuepress

    在开始搭建博客前，先来介绍一下 Vuepress，Vuepress 是由 Vue 支持的静态网站生成工具，Vue 现在很火大家都知道或者使用过吧，一种轻量级且好用的框架，因为 Vue 上手起来很简单，所以 Vuepress 使用起来也不难。如果想快速搭建一个静态的博客网站来简单记录记录笔记或者文章，用 Vuepress 是个不错的选择，因为它对新手很友好，而且手机移动端和电脑 PC 端都是可以查看的

    前提条件 VuePress 需要 Node.js >= 8.6

    # 基础版

    介绍完 Vuepress，我们就可以开始进行博客的搭建了。先给大家看一下最终完成之后的博客项目的目录结构吧，我用的是WebStrom。

    # 1.本地初步搭建

    在你自己的worksapce目录下创建一个文件夹，我就叫myblog了，就是你项目的根目录。

    mkdir blogs
cd blogs
    1
    2

    然后使用 npm 或者 yarn 进行初始化，我这里使用的是 npm 包管理器。

    npm init -y
    1

    然后安装 VuePress 为本地依赖。

    npm install -D vuepress
    1

    完成这些之后开始创建我们的第一篇文档，先创建 docs 文件夹，然后在文件夹下创建 README.md 文件，README.md 文件中我们可以随意加入点内容，比如 Hello Vuepress。

    这里需要注意，如果使用如下命令创建的话，后续可能会出现乱码的情况，这时候只需将README.md 的编码格式改为 UTF-8 即可

    mkdir docs 
echo '# Hello VuePress' > docs/README.md

    2

    最后，我们再向 package.json 中的 scripts 里添加如下代码。

    "scripts": 
{ 
 "docs:dev": "vuepress dev docs",  
 "docs:build": "vuepress build docs" 
}

    # 整体样式修改

    我们可以通过添加相关文件来修改博客的默认样式。

    # I. 主题色修改

    VuePress 默认主题颜色是绿色，我们可以将主题颜色修改为自己喜欢的颜色。在 .vuepress 目录下创建 styles 文件夹，styles 文件夹里创建一个 palette.styl 文件，文件代码如下：

    $accentColor = #3178c6  // 蓝色主题色
    1

    这就可以将主题颜色改为蓝色了。

    # II. 其他样式修改

    如果我们切换到暗黑模式下，可能会发现文字看不清楚，所以我们需要修改文字的颜色和背景色。

    在之前创建好的 styles 文件夹下再新建一个 index.styl 文件，文件代码如下：

    .dark .content__default code {
  background-color: rgba(58,58,92,0.7);
  color: #fff;
}

    之所以称为基础版，是因为我们并没有添加额外的插件来丰富博客。如果想要美化升级自己的博客，接下来我将介绍通过插件来丰富后的升级版博客。

    # b.文章信息

    文章信息包括很多，比如标题，日期，分类，标签等等，实现这些我们只需要在每篇 md 文档的前面加入如下示例代码即可。

    title: 2023-5-4 青年节的开始🔥
date: 2023-5-4
categories:
  - 前端
tags:
  - 记录
sticky: 1

    title 是标题，date 是日期，categories 是分类，tags 是标签，可以有多个分类和标签，sticky 可以让文章置顶，数字表示顺序，1 表示在最上面，2 其次。这里的代码格式一定是按照如上写法：变量后面要加冒号，冒号后面要有空格，不要加逗号。

    这里的作者名 ShowedMaker是在 config.js 中的 themeConfig 里设置的，这样每篇文章里就会附上设置好的作者名。

    author: "ShowedMaker"

    # 部署

    # a.在GitHub上创建一个仓库

    # b.一个命令行发布到github上：

    在 docs/.vuepress/config.js 中设置正确的 base:

    如果你打算发布到 https://<USERNAME>.github.io/，则可以省略这一步，因为 base 默认即是 "/"。

    如果你打算发布到 https://<USERNAME>.github.io/<REPO>/（也就是说你的仓库在 https://github.com/<USERNAME>/<REPO>），则将 base 设置为 "/<REPO>/"。

    module.exports = {
  base: '/vuepress-starter/', 
}
    1
    2
    3

    # c.创建脚本文件:

    在project的根目录下，创建一个deploy.sh文件：注意 需要将 USERNAME 和 REPO 替换成自己的GitHub账号和仓库名称

    #!/usr/bin/env sh
# 确保脚本抛出遇到的错误
set -e
# 生成静态文件
npm run docs:build
# 进入生成的文件夹
cd docs/.vuepress/dist
# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME
git init
git add -A
git commit -m 'deploy'
# 如果发布到 https://<USERNAME>.github.io  USERNAME=你的用户名 
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master
# 如果发布到 https://<USERNAME>.github.io/<REPO>  REPO=github上的项目
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages
cd -
